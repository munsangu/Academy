Javascript의 변수(Variable)

모든 프로그래밍 언어는 데이터 참조하기 위해 변수를 사용
이 때, 참조하는 데이터가 변할 수 있어서 변수 ex) let a = "Hello" let a = 200 console.log(a) // 200

Javascript의 변수와 상수

  • var : Javascript 태초의 변수, 함수 지향 언어의 특징을 가진 블럭이 모호한 상태(사용하지 않음)
  • let : ES6에 탄생한 변수, Class 추가와 함께 블럭의 구분을 명확하게 함(var 대신 사용)
  • const : 변수가 아닌 상수, 선언과 동시에 데이터를 참조해야 하고 그 데이터를 변경하는 것은 안됨
    참조하는 데이터의 상태를 변경할 수 있음

변수의 식별자(이름)

프로그래밍에서 식별자 즉, 이름 짓기는 중요 대체로 유지 보수에 용의하기 위해 약속된 방식이지만 이름이 오류를 만들 수 있어, 오류가 있는 코드를 배포하는 Javascript 특성상 이름 짓기가 더 중요 ex) let var = 10; (예약어를 변수로 사용한 오류)

  • 변수 이름은 문자, 숫자, 밑줄, $가 포함될 수 있음
  • 변수는 숫자로 시작하는 것을 권장하지 않음 ex) let 1num; (X) let num1; (O)
  • 변수의 이름은 구체적이어야하고 문자와 문자의 결합을 camelCase, snake_case, PascalCase로 함
    ex) memNum, userId / mem_num, user_id / Mem_Num, User_Id
  • 상수(const)의 이름은 모두 대문자이고 _표기함(ex) COLOR_RED)
  • 이름은 대소문자를 구분(num !== Num !== NUM)
  • 예약어는 변수 이름으로 사용할 수 없음(function, var, let, Number)

식별자에 그 밖의 규칙(권장 사항)

  • 읽고 이해할 수 있는 이름을 사용(userName, userId, colorName ...)
  • a, b, c, i와 같은 짧은 이름은 사용하지 않는 것이 좋음 단, 구조적으로 변수명이 명확할 때에는 사용 가능
    ex)for(let i = 0 ; i < 5 ; i++ ){}(i = index)
  • 소속된 팀의 규칙을 따를 것(userID, memberNum) (X) / (userId, userNum) (O)
