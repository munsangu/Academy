javascript class

  • 클래스 class : 자동차를 조작=> 엑셀과 브레이크로 속도를 조작, 핸들로 방향을 조작, 직접 엔지에 기름(x)=> 캡슐화
  • new object()=>object={name:"최경민", age: 30}=>object={name:null, age: -1}
  • const arr=[1,2,3];(x 리터럴한 선언)=>new Array(1,2,3)(배열객체를 생성하는 실제방식
  • class Person{name,age; constructor(n,a){ (n)?this.name=n:"이름없음"; (a>=0)this.age=a:0;} }=> new Person("",-1); => {name:"이름없음",age:0}
  • class 를 선언하는 이유1(캡슐화): 사용할 객체를 사용하지 않는것과 분리(은닉), 사용하는 객체를 문제가 생기지 않도록 처리, (속성)멤버변수를 선언하고 (행위)메서드를 선언하는 행위
  • 언어적 측면에서 접근 지정자에 의해 정의된 변수나 멤버 변수에 의해 오류가 발생할 것을 사전에 방지하는 행위(name=null 기계적 오류 X)
  • "최_경민".split("_"), null.split("_")=>오류
  • class 를 선언하는 이유2(재사용): 상속으로 class를 재사용 가능
  • class 를 선언하는 이유3(유지보수): 선언되 class를 수정하면 참조하는 모든 class가 변경된다.(유지보수가 용이)
  • class 의 단점1: 설계시 개발 속도를 느리게 한다.(개발자의 능력)
  • class 의 단점2: 객체가 많고 설계가 잘못되면 용량이 커진다.(개발자의 능력)
  • class 의 단점3: 자바스크립트는 class를 function의 대용품으로 사용중 함수가 클래스인 척 하고 있다. (모든 브라우저에 동작X,자바스크립트에 대해 깊이있는 이해가 필요)
