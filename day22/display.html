display 속성으로 node의 레이아웃 제어

display block

항상 새로운 라인으로 시작하고 가로폭 전체를 차지함(default => width: 100%)

test
  • width가 지정될 경우, margin이 남은 가로폭을 채움
  • (default) margin-left: 0, margin-right: auto로 왼쪽 정렬됨
  • 보통 block은 inline 요소를 포함하기 위해 지정
  • box model의 속성을 모두 포함
    • width / max-width / min-width
    • height / max-height / min-height
    • padding / padding-(top, right, bottom, left)
  • margin / margin-(top, right, bottom, left)
  • border / border-(top, right, bottom, left)

display inline

줄을 바꾸지 않고 다른 요소와 한 행에 위치함(default => width: fit-content)

test
  • box 모델 중에 width, height, margin-top / bottom을 지정할 수 없음
  • height 대신 line-height로 상하 여백을 지정
  • inline의 스타일을 변경하거나 쓰임을 명시하기 위해 작성

display inline-block

inline과 같이 줄을 바꾸지 않고 다른 요소와 한 행에 위치하나, block처럼 box 모델을 모두 지정할 수 있음

test
  • 요소 내부 컨텐츠의 위쪽 여백을 4px로 자동 지정함

display none

  • display none은 해당 요소를 화면에 나타내지 않음
  • 해당 요소는 node 개체가 삭제되지 않음

visibility hidden

  • visibility: hidden은 해당 요소를 보이지 않게 함
    • 해당 요소의 box 속성을 그대로 남기고 보이지만 않게 됨
    • table의 열이나 행이 사라지지 않을 때는 collapse나 none을 사용해야 함
  • visibility: visible 은 default 값으로 해당 요소를 보이게 함

opacity

  • opacity = 0 은 visibility = hidden과 거의 유사
  • opacity는 요소의 투명도를 조절
  • rgba로 요소의 배경 투명도를 변경하는 것과는 차이가 있음

transform scale(x, y)

  • style.transform은 node 객체를 2D 또는 3D로 변환해서 회전, 배율, 이동, 기울기 등을 변경함
  • 실제 크기가 변경되는 것이 아니라 스냅샷 같은 것이 변경되어 화면에 표현되기 때문에
  • 애니메이션 효과를 적용하기에 최적
